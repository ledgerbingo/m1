Vision
AI agents that can pay for the world.

DEO is an execution layer that lets an automated agent unlock a paid API endpoint by presenting a cryptographic proof of authorization. In production, the proof is created by a transaction that transfers stable-value funds to the service. For reliability here, the site runs in a simulation mode while preserving the exact HTTP flow.

How the product works (end-to-end)
Service side: A paid endpoint answers unauthenticated requests with a 402 response and a challenge header. That header tells a client where to verify and what kind of proof is expected.
Agent side: The agent reads the challenge, generates the proof (in production: by submitting an authorization action), and retries the original request with the proof attached.
Verification: The verifier checks the proof and returns OK. Once verified, the endpoint serves the premium payload immediately.
Why this matters: You get a clean bridge between modern HTTP services and autonomous software operators, enabling pay-per-call APIs that feel instant for agents.
Concrete example
Endpoint: GET /weather returns premium weather data.
First request: the server returns 402 plus WWW-Authenticate. This is the machine-readable challenge your agent uses to decide what to do next.
Second request: the agent retries with X-Payment-Proof. If the proof verifies, the server responds with the premium JSON payload.
